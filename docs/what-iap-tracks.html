What IAP Tracks

In the current beta, IAP tracks and certifies a narrow set of cryptographic facts. It does not capture the full semantic meaning of your agent.

  • Agent identity bindings (public key to agent_id).
  • Ordered state checkpoints (memory_root, sequence).
  • Identity-anchor and continuity certificate artifacts.
  • Declared lineage transitions and key-rotation transitions where those certificate types are used.
  • Signed registry artifacts verifiable offline.

In practice, the current beta flow records local state into AMCS first, then certifies the resulting memory_root and sequence.
