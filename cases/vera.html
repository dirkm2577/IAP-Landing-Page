Case Study

VeraConfiguration Commitments

Vera needs configuration stability: changes should be explicit, time-stamped, and auditable. The objective is detectable declared configuration drift via hash commitments—not runtime enforcement.

IAP verifies identity, declared lineage, and a time-sequenced chain of signed commitments (often hashes of internal state). It does not verify behavior, semantic correctness, or prevent runtime compromise.

IAP Integration

  • Commit a canonical manifest: Deterministically serialize the agent configuration manifest and hash it.
  • Continuity: Anchor the manifest hash as a continuity commitment on a defined cadence (or upon change).
  • Audit: Observers compare commitments over time to detect when declared configuration changed.

What IAP Guarantees

  • Time-stamped declared configuration commitments.
  • Detectable changes in committed configuration hashes across time.
  • Offline verifiable audit trail of configuration commitments.

What IAP Does Not Guarantee

  • Preventing prompt injection, runtime compromise, or tool abuse.
  • Guaranteeing the runtime always obeyed the configuration.
  • Behavioral determinism.
